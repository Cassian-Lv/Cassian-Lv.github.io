1.了解算法复杂度

O(log n)

  1. 对数增长：O(log n)表示算法的运行时间（或空间消耗）与输入规模n的对数成比例。具体来说，如果一个算法的时间复杂度是O(log n)，那么随着输入规模n的增加，所需的资源增长速度非常缓慢。
  2. 举例说明：假设一个算法的时间复杂度是O(log n)，那么当n增加时，算法的性能提高得非常显著。例如，如果n从100增加到1000，O(log n)的算法可能只需要稍微增加一点时间，因为log 1000 ≈ 3，而log 100 ≈ 2。
  3. 常见应用：O(log n)的复杂度通常出现在某些分治算法（如二分搜索）、树结构（如二叉搜索树、平衡二叉树）的操作等场景中。这些算法通常能够快速地缩小问题的规模，并且以对数级别的速度处理数据。
  4. 与其他复杂度比较：与O(n)（线性复杂度）相比，O(log n)增长更为缓慢；而与O(1)（常数复杂度）相比，O(log n)则随着n的增加而略微增加。

总之，理解O(log n)可以帮助你意识到某些算法在处理大规模数据时的高效性和优越性，因为它表明算法的性能随着数据量的增加而不会呈现线性增长。

算法中前缀和

算法中的前缀和（Prefix Sum）指的是一个数组中从第一个元素开始到指定位置之间所有元素的和。这个概念在很多算法问题中都很有用，特别是涉及区间和查询的情况下。

具体来说，对于一个数组 ( \text{arr} )，其前缀和数组 ( \text{prefix}[i] ) 定义为： [ \text{prefix}[i] = \sum_{k=0}^{i} \text{arr}[k] ]

换句话说，( \text{prefix}[i] ) 存储了数组 ( \text{arr} ) 中从下标 ( 0 ) 到 ( i ) 的所有元素的累积和。

前缀和的主要优势在于，一旦计算出前缀和数组，我们可以在 ( O(1) ) 时间内回答任意区间 ( [l, r] ) 的元素和查询，而不必重新计算每次查询的和，这样可以大大提高效率，特别是在需要频繁查询区间和的情况下。

前缀和也常用于解决一些子数组和问题，例如最大子数组和、子数组和为特定值的个数等。

平衡二叉树

平衡二叉树（Balanced Binary Tree）具有以下几个主要特点：

  1. 高度平衡性：平衡二叉树是一种特殊的二叉搜索树（BST），它保证了左右子树的高度差不超过1。这确保了树的高度相对较小，使得基本的操作（如查找、插入、删除）在平均情况下的时间复杂度为 O(log n)，其中 n 是树中节点的数量。
  2. 结构特性：除了保持高度平衡外，平衡二叉树的每个节点都具有左子树和右子树，且左子树和右子树也都是平衡二叉树。
  3. 查找效率高：由于树的平衡性，平衡二叉树的查找操作非常高效。在最坏情况下，它的查找时间复杂度为 O(log n)，这是因为树的高度始终保持在 log n 的范围内。
  4. 插入和删除操作：插入和删除操作都会导致平衡二叉树的结构发生变化，但是通过旋转（如左旋、右旋、双旋等）操作来维持树的平衡性。这些操作保证了插入和删除的时间复杂度也是 O(log n)。
  5. 应用领域：由于平衡二叉树在动态数据集合中具有高效的性能特点，它常被用作底层数据结构，用来实现各种高级数据结构，如集合、映射（例如红黑树就是一种平衡二叉树）、有序集合等。

总之，平衡二叉树通过维护树的平衡性，使得在插入、删除和查找操作时能够保持较高的效率，这是它与普通二叉搜索树最显著的区别之一。

java中的左移运算和右移运算

在Java中，>> 和 << 是位运算符，用于对整数进行位移操作。

  1. **>>**** 右移运算符**：
    • >> 用于将一个数的所有位向右移动指定的位数，并用原来的最高位填充移动后左侧空出的位。
    • 对于有符号整数，右移操作会保留符号位（即最高位）。例如，如果最高位是1，则右移后左侧空位会用1填充；如果最高位是0，则用0填充。
  • **<<**** 左移运算符**：
  • << 用于将一个数的所有位向左移动指定的位数，并用0填充右侧空出的位。
  • 左移操作会导致数值成倍增加。例如，左移1位相当于乘以2，左移2位相当于乘以4，依此类推。

这些位移操作在Java中通常用于对位级别的操作，例如优化算法、处理位掩码、或者对于特定的硬件接口和数据结构。

按位异或运算的特性是，如果两个对应的二进制位相同，则结果为0；如果不同，则结果为1。

现在我们逐步进行运算：

  1. 初始值设为0： 0
  2. 异或 2： 0 ^ 2 = 2
  3. 异或 4： 2 ^ 4 = 6
  4. 异或 6： 6 ^ 6 = 0
  5. 异或 8： 0 ^ 8 = 8

因此，0 ^ 2 ^ 4 ^ 6 ^ 8 的结果不是0，而是8。

所以，按位异或运算 0 ^ 2 ^ 4 ^ 6 ^ 8 的输出结果是 8。

二进制计算

计算一个十进制数的二进制表示通常可以通过除以2并记录余数的方法来实现。以下是将十进制数转换为二进制的步骤：

  1. 除以2并记录余数：
    • 将要转换的十进制数除以2，记录下商和余数。
    • 商继续除以2，再次记录商和余数，直到商为0为止。
  2. 反向排列余数：
    • 将每次记录的余数按照计算顺序从下往上排列，这样得到的就是该十进制数的二进制表示。

举例来说，我们将十进制数 25 转换为二进制：

  • 25 除以 2 得商 12，余数 1
  • 12 除以 2 得商 6，余数 0
  • 6 除以 2 得商 3，余数 0
  • 3 除以 2 得商 1，余数 1
  • 1 除以 2 得商 0，余数 1

将上述余数反向排列，得到的二进制表示为 11001。

另一种方法是使用位运算，从最低有效位（最右边的位）开始逐步获取每一位的值。这在编程中是常见的做法，可以通过按位与（&）和移位操作来提取二进制位。

int类型的二进制表示

在计算机中，整数数据（比如 Java 中的 int 类型）是以二进制形式存储的。每个整数都被分解成一个二进制数字序列，其中每一位可以是0或1。

为什么 int a = 12; 的二进制表示有这么多位呢？这是因为在Java中，int 类型通常使用32位来存储一个整数。这种长度是一种标准，被称为固定长度的二进制表示。下面是解释：

  1. 固定长度的二进制表示：
    • 在计算机科学中，整数类型（如 int）通常具有固定的位数来存储数值。在Java中，int 类型使用32位（也就是4个字节）来存储整数。
    • 32位二进制可以表示的整数范围是从 -2^31 到 2^31 - 1，即大约 -2.1 billion 到 2.1 billion 的整数范围。
  2. 二进制表示的具体形式：
    • 当你写 int a = 12; 时，计算机将十进制数12转换为32位二进制数。在这个例子中，12 的二进制表示是 00000000 00000000 00000000 00001100。
    • 每一个0或1代表一个比特（bit），每8个比特组成一个字节（byte）。因此，32位二进制就是4个字节。
  3. 存储和处理：
    • 计算机以二进制形式存储数据，这样可以更有效地进行处理和传输。固定长度的二进制表示有助于确保每个整数在内存中占据相同的空间，这对于编程和系统设计非常重要。

总结来说，Java中整数的二进制表示有32位，是一种标准化的存储方式，旨在提供高效的数据处理和存储。

源码、反码、补码

源码、反码和补码是计算机中表示有符号整数的不同方式，主要用于二进制数值的表示。下面是这些术语的定义以及它们之间的关系：

  1. 源码（原码）

定义: 源码是指将一个整数转换成二进制后，最高位作为符号位（0表示正数，1表示负数），其余位表示数值的绝对值。

示例: 数字 5 的二进制表示为 0000 0101（假设字长为 8 位）。

  1. 反码

定义: 对于正数，反码与源码相同；对于负数，反码是将源码除了符号位之外的所有位取反（0变1，1变0）。

示例: 数字 -5 的源码为 1000 0101，取反后得到反码 1111 1010。

  1. 补码

定义: 对于正数，补码与源码相同；对于负数，补码是在反码的基础上再加 1。

示例: 数字 -5 的反码为 1111 1010，再加 1 得到补码 1111 1011。

  • 支持随机访问：数组允许在
  • 时间内访问任何元素。
  • 缓存局部性：当访问数组元素时，计算机不仅会加载它，还会缓存其周围的其他数据，从而借助高速缓存来提升后续操作的执行速度。

连续空间存储是一把双刃剑，其存在以下局限性。

  • 插入与删除效率低：当数组中元素较多时，插入与删除操作需要移动大量的元素。
  • 长度不可变：数组在初始化后长度就固定了，扩容数组需要将所有数据复制到新数组，开销很大。
  • 空间浪费：如果数组分配的大小超过实际所需，那么多余的空间就被浪费了。

数组的应用

数组是一种基础且常见的数据结构，既频繁应用在各类算法之中，也可用于实现各种复杂数据结构。

  • 随机访问：如果我们想随机抽取一些样本，那么可以用数组存储，并生成一个随机序列，根据索引实现随机抽样。
  • 排序和搜索：数组是排序和搜索算法最常用的数据结构。快速排序、归并排序、二分查找等都主要在数组上进行。
  • 查找表：当需要快速查找一个元素或其对应关系时，可以使用数组作为查找表。假如我们想实现字符到 ASCII 码的映射，则可以将字符的 ASCII 码值作为索引，对应的元素存放在数组中的对应位置。
  • 机器学习：神经网络中大量使用了向量、矩阵、张量之间的线性代数运算，这些数据都是以数组的形式构建的。数组是神经网络编程中最常使用的数据结构。
  • 数据结构实现：数组可以用于实现栈、队列、哈希表、堆、图等数据结构。例如，图的邻接矩阵表示实际上是一个二维数组。

链表

数组和链表的时间复杂度分析

    链表中删除和添加元素的时间复杂度确实是 O(1)，但这有一个前提条件：你需要知道要删除或添加的元素的 前驱节点 的引用。这是因为链表中的元素是通过指针连接的，删除或添加一个节点只需要修改其前驱节点的指针即可。

查找时间复杂度

    查找一个元素在链表中的位置确实需要遍历链表，时间复杂度为 O(n)。这是因为链表中的元素不是连续存储的，而是通过指针连接在一起，因此需要从头节点开始逐个节点地遍历直到找到目标元素或到达链表的末尾。

删除和添加时间复杂度

    一旦你知道了要删除或添加的元素的前驱节点，删除和添加操作就可以在常数时间内完成，即时间复杂度为 O(1)。

删除操作：

     如果你已经有了待删除节点的前驱节点的引用，你可以简单地修改前驱节点的指针，使其指向待删除节点的下一个节点。

这样做不需要遍历整个链表，因此时间复杂度为 O(1)。

添加操作：

      同样，如果你已经有了待插入位置的前驱节点的引用，你可以简单地修改前驱节点的指针，使其指向新节点，再让新节点的指针指向原来前驱节点指向的节点。

这个操作也不需要遍历整个链表，因此时间复杂度为 O(1)。

总结

查找操作：在链表中查找一个元素的时间复杂度为 O(n)。

删除和添加操作：一旦你知道了待删除或添加元素的前驱节点，删除和添加操作的时间复杂度为 O(1)。

因此，当你谈论删除和添加的时间复杂度时，通常是指在已知前驱节点的情况下。如果需要先找到前驱节点，则还需要考虑查找的时间复杂度。
